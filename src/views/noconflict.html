No-Conflict

In case of using the UI framework on top of a current .dw- class setup, there's the option of a no-conflict alternative package of all styles. Class prefixes change from .dw- to .dw-nc-.

UI Reset

.dw-ui-reset intializes a very basic UI reset to prevent former styles affecting into new components using the no-conflict styles. Always initialize a screen or a single component with the .dw-ui-reset helper to set common base values for color, fonts and box-sizing:

<div class="dw-ui-reset dw-nc-modal" role="dialog">
  ...
</div>

Wizard

Navigating within a Wizard

There are two kinds of navigational options for a Wizard.

  • progress bar
  • previous & next buttons

Navigating via previous & next buttons should always be a linear one-step navigation. Next leads one step forward, Previous leads one step back from the users current point in the progression. There’s no skipping steps when using these buttons. If a change requires the next step in the progression to be unavailable, the button navigation will lead the the next / previous available step. Navigation with the progress bar does allow skipping steps, if possible from a data set change perspective.

Hiding & Disabling Progression Steps

When to hide or when to disable progressional steps? If a step within the progression becomes obsolete remove/hide it from the progress bar. Disabled steps should be used to preview a complete progression.
