Release Highlights

Whats New in Commerce Cloud: Release 18.5.

Language Analyzer for Site Search
A language analyzer understands the characteristics of a specific language, such as foreign language characters, and increases the relevancy of search results. New filters have been added to the language analyzer in Commerce Cloud to further increase the relevancy of site search results. In addition, the language analyzer will be available for four new languages: Brazilian Portuguese, Sorani Kurdish, Irish, and Hindi.

XML Sitemap Enhancements
Maximize search engine ranking and increase traffic to your site with two enhancements to XML Sitemaps:
1. Merchants gain more control over their URL taxonomy by excluding category or folder level information. This helps brands increase rankings & organic traffic with URLs that are easier for search engines to read.
2. When updating the alias hostname in Business Manager, changes are now refreshed instantly in the XML sitemaps menu. Previously, users would have to wait one minute to see their changes reflected.

OCAPI Explorer Enhancement Debugger Information
The Open Commerce API (OCAPI) Debugger is an extension of the OCAPI Explorer and allows developers to easily troubleshoot OCAPI requests, and provides detailed information about the execution path across all API endpoints. This allows developers to efficiently design and test storefront extensions and build unique digital experiences.

Real-time metrics for Reports & Dashboards (Open Beta)
A new set of real-time metrics have been added to Reports and Dashboards giving business users easy-to-read dashboards that offer insight into revenue, top selling products, profitability of promotions, and more. This includes the ability to see metrics for a single site or across all sites and filter by customer type, site and product to get granular insight into your business. 

 

 

 

 

 

 

BM Release Updates & Notifications

Updated: Meta API Change 

Last year we announced the 17.3 Commerce Cloud release included enhancements to the Meta API method in order to strengthen its overall functional and security profile. We asked our customers to take advantage of the new API and begin updating client code with the new API in the 17.3 Commerce Cloud release.

We have delayed disabling the old API a little longer to allow customers a little more time to complete their update to the new API. We will not disable the old API with the 18.5 release as previously indicated.

However, we are asking customers to please make your best efforts to update your client code to work with the latest Meta API. This is for your benefit as the new API includes security and functional improvements. 
We will update this communication within the next few weeks to indicate a new target release where we will disable the old API.

Information about the new Meta API (and how to access it) is described in this article  https://documentation.demandware.com/DOC1/topic/com.demandware.dochelp/OCAPI/18.3/usage/Metadata.html?cp=0_12_2_20

Here are some differences between the former Meta API and the new one:

* The customer needs a valid OAuth token to access the new Meta API.The old Meta API did not require any authentication.

* The new Meta API has a slightly different API path. Instead of /dw/meta/rest/ the new Meta API must be accessed via /dw/meta/v1/rest/

* The new Meta API must be accessed securely, i.e. via https

If you have any questions about this change, please contact your Success Manager or Technical Account Manager for further assistance.
